# WGCP+CTM 消融实验方案（服务器版）

## 1. 目的与边界

本方案只回答三个问题：

1. 低频锚点来源（`LL_orig` vs `LL_hat`）是否影响语义稳定性。
2. 高频策略（`pred` vs `orig` vs `fused`）对恢复率与细节保真的影响。
3. 自校正循环（loop）是否带来真实增益。

注意：

- 数据建议使用 `data/imagenet_real` + `--glob "*.JPEG"`。
- `ablation_ll_source` 与 `ablation_hard_hf_source` 是“仅用于消融”的开关，不是主线默认配置替代品。

## 2. 消融矩阵（A0-A5）

| Case | 配置变化 | 作用 |
|---|---|---|
| A0_baseline | `ll=orig, hard_hf=pred, self_correct_k=1` | 基线 |
| A1_ll_hat | `ablation_ll_source=hat` | 检查低频锚点漂移 |
| A2_hard_hf_orig | `ablation_hard_hf_source=orig` | 检查保留原始高频的代价 |
| A3_ll_hat_hf_orig | A1+A2 同时启用 | 压力测试 |
| A4_fused | `replacement_mode=fused` | 检查软融合折中 |
| A5_no_loop | `self_correct_k=0` | 检查 loop 贡献 |

## 3. 一键执行

```bash
cd /home/you/CAMP
bash scripts/run_ablation_ctm_server.sh ./data/imagenet_real ./outputs/wgcp_ablation_ctm
```

可选环境变量：

```bash
MAX_IMAGES=100 \
MIN_CLEAN_CONF=0.05 \
SAVE_DETAIL_EVERY=10 \
PATCH_MODE=0 \
HF_PRESERVE=0.35 \
HF_SHRINK=0.6 \
GLOB_PATTERN="*.JPEG" \
TORCH_CACHE_DIR=/home/you/.cache/camp_torch \
bash scripts/run_ablation_ctm_server.sh ./data/imagenet_real ./outputs/wgcp_ablation_ctm
```

存储建议（正式 100 张时推荐）：

1. `SAVE_DETAIL_EVERY=10` 表示每 10 张只保存 1 张的详细中间结果图（约 10 组）。
2. `summary.json` 仍会保留全部 100 张样本的指标记录，不影响统计。
3. 如需整套切换为 Patch-WGCP，可设置 `PATCH_MODE=1`，并配合 `PATCH_STRIDE=32`、`PATCH_LOWFREQ_ALPHA=0.1` 作为首轮配置。

附加参数透传给每个 case：

```bash
bash scripts/run_ablation_ctm_server.sh ./data/imagenet_real ./outputs/wgcp_ablation_ctm \
  --device cuda --archive_dir /home/you/.camp_runs/CAMP
```

## 4. 输出文件

- 每个 case：`outputs/wgcp_ablation_ctm/<case>/summary.json`
- 总览聚合：`outputs/wgcp_ablation_ctm/ablation_overview.json`

`ablation_overview.json` 包含关键聚合指标，可直接用于表格整理。

## 5. 判读顺序（推荐）

1. `A0` vs `A5`：先确认 loop 是否真正有收益。
2. `A0` vs `A1`：确认低频锚定 `LL_orig` 的价值。
3. `A0` vs `A2/A4`：比较细节保真与恢复率折中。

核心指标：

- `recover_rate_on_attacked`（越高越好）
- `clean_pred_consistency_rate`（越高越好）
- `clean_vs_purified.SSIM/PSNR`（越高越好）
- `NFE` 与 `single_step_infer_ms`

## 6. 与主流程关系

消融是分析工具，不是日常默认运行入口。

日常主流程建议使用：`scripts/run_wgcp_ctm_server.sh`，并优先参考 `docs/CTM服务器一键配置与结合指南.md` 的推荐配置。
