# WGCP+CTM 消融实验方案（服务器版）

## 1. 实验目标

本文档用于回答三个核心问题：

1. 当前默认实现（CAMP/WGCP 主线）是否是更稳的折中点。
2. 低频锚点来源（`LL_orig` vs `LL_hat`）对语义稳定性的影响有多大。
3. 高频来源与自校正循环对“恢复率-细节保真”权衡的贡献。

重要：消融默认数据集使用 `data/imagenet_real`（`--glob "*.JPEG"`）。
不要使用 `data/clean_samples` 作为正式消融基准，以避免明显 Domain Shift 导致的低置信度与高方差问题。

默认主线保持不变：

- `ablation_ll_source=orig`
- `replacement_mode=hard`
- `ablation_hard_hf_source=pred`
- `self_correct_k=1`

## 2. 消融矩阵（A0-A5）

| Case | 变更项 | 目的 |
|---|---|---|
| A0_baseline | 主线默认 | 作为全部对照基线 |
| A1_ll_hat | `ablation_ll_source=hat` | 检验低频锚点改为预测低频后的语义漂移 |
| A2_hard_hf_orig | `ablation_hard_hf_source=orig` | 检验 hard 模式保留原始高频对细节/净化的影响 |
| A3_ll_hat_hf_orig | 同时启用 A1+A2 | 压力测试，观察极端组合稳定性 |
| A4_fused | `replacement_mode=fused` + `hf_preserve/hf_shrink` | 检验软融合是否更好平衡细节与恢复率 |
| A5_no_loop | `self_correct_k=0` | 检验自校正（bridge）模块的增益 |

## 3. 一键运行（推荐）

前置条件：

1. 已完成 CTM 服务器配置，存在 `configs/ctm_server_config.json`。
2. 已在服务器上激活并可使用 `camp` conda 环境。
3. 输入图片目录已准备好（建议 `data/imagenet_real`，并使用 `--glob "*.JPEG"`）。

执行：

```bash
cd /home/HHY/CAMP
bash scripts/run_ablation_ctm_server.sh ./data/imagenet_real ./outputs/wgcp_ablation_ctm
```

可选环境变量：

```bash
MAX_IMAGES=100 \
MIN_CLEAN_CONF=0.05 \
HF_PRESERVE=0.35 \
HF_SHRINK=0.6 \
GLOB_PATTERN="*.JPEG" \
TORCH_CACHE_DIR=/home/HHY/CAMP/.cache/torch \
bash scripts/run_ablation_ctm_server.sh ./data/imagenet_real ./outputs/wgcp_ablation_ctm
```

额外参数透传给每个 case（示例）：

```bash
bash scripts/run_ablation_ctm_server.sh ./data/imagenet_real ./outputs/wgcp_ablation_ctm \
  --device cuda --archive_dir /home/HHY/.camp_runs/CAMP
```

## 4. 输出目录说明

根目录：

- `outputs/wgcp_ablation_ctm/A0_baseline/summary.json`
- `outputs/wgcp_ablation_ctm/A1_ll_hat/summary.json`
- `...`
- `outputs/wgcp_ablation_ctm/ablation_overview.json`

每个 case 下单样本目录仍保留完整中间结果图，包括：

- `LL_orig`
- `LL_hat`
- `LL_anchor`（本次真正用于重建的低频）
- 高频可视化与最终净化图

## 5. 如何判读结果

核心看三类指标：

1. 防御有效性：
   - `recover_rate_on_attacked` 越高越好。
   - `clean_pred_consistency_rate` 越高越好。
2. 图像保真：
   - `clean_vs_purified.SSIM` 越高越好。
   - `clean_vs_purified.PSNR` 越高越好。
3. 代价：
   - `NFE` 与 `single_step_infer_ms`。

建议判读顺序：

1. 先确认 A0 是否比 A5 好（验证 self-correct 有效）。
2. 再比较 A0 vs A1（验证 `LL_orig` 锚点价值）。
3. 比较 A0 vs A2/A4（验证高频策略是否改善细节且不牺牲恢复率）。

## 6. 手工复现实验（单 case）

如果你只想先跑一个 case（例如 A1）：

```bash
cd /home/HHY/CAMP
export PYTHONPATH=$PYTHONPATH:$(pwd)/experiments:$(pwd)

python experiments/wgcp_attack_eval.py \
  --input_dir ./data/imagenet_real \
  --glob "*.JPEG" \
  --output_dir ./outputs/wgcp_ablation_ctm/A1_ll_hat_manual \
  --attack pgd --eps 0.0313725 --pgd_steps 10 --pgd_alpha 0.0078431 \
  --classifier resnet50 \
  --predictor_type module \
  --predictor_module experiments.ctm_adapter_sony:CTMRepoPredictor \
  --predictor_kwargs_json '{"ctm_repo": "/home/HHY/CAMP/third_party/ctm", "checkpoint": "/home/HHY/CAMP/.cache/ctm/ema_0.999_049000.pt", "class_cond": true, "class_label": 0, "predictor_image_size": 64}' \
  --predictor_image_size 64 \
  --t_star 40 --t_bridge 10 --self_correct_k 1 \
  --replacement_mode hard \
  --ablation_ll_source hat \
  --ablation_hard_hf_source pred \
  --min_clean_conf 0.05
```

建议流程：

1. 烟雾测试：`MAX_IMAGES=6` 快速确认脚本可跑通。
2. 正式消融：`MAX_IMAGES=50~100`（推荐 100）再出表格，降低方差并提升统计显著性。

## 7. 建议记录模板（论文表格）

建议把 `ablation_overview.json` 汇总成表格列：

1. `Case`
2. `attack_success_rate`
3. `recover_rate_on_attacked`
4. `clean_pred_consistency_rate`
5. `mean(clean_vs_purified.SSIM)`
6. `mean(clean_vs_purified.PSNR)`
7. `NFE`
8. `single_step_infer_ms`

这样可以直接对应论文中的 “Effectiveness / Fidelity / Efficiency” 三轴分析。
