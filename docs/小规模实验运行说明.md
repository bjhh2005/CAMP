# WGCP 小规模实验运行说明

## 1. Conda 环境管理（替代 requirements.txt）

```bash
conda env create -f environment.yml
conda activate camp
```

如果环境已存在，使用更新命令：

```bash
conda env update -n camp -f environment.yml --prune
```

当你新增/删除依赖后，用 `conda export` 回写环境文件：

```bash
conda export -n camp --from-history --file environment.yml
```

仓库里也提供了导出脚本：

```bash
# Linux
bash scripts/export_conda_env.sh camp environment.yml

# Windows PowerShell
powershell -ExecutionPolicy Bypass -File scripts/export_conda_env.ps1 -EnvName camp -OutputFile environment.yml
```

## 2. 仅跑净化（已有对抗图）

将对抗图放到 `data/adv_samples/` 后运行：

```bash
python experiments/wgcp_small_scale.py \
  --input_dir data/adv_samples \
  --output_dir outputs/wgcp_debug \
  --max_images 4 \
  --t_star 40 \
  --t_bridge 10 \
  --self_correct_k 1 \
  --replacement_mode hard
```

## 3. 一体化流程（推荐）

如果你现在没有被污染图片，也没有分类器，可以直接跑下面脚本：

- 使用 ImageNet 预训练分类器（自动下载权重）
- 对 clean 图生成对抗样本（FGSM/PGD）
- 执行 WGCP 净化
- 输出分类行为与图像质量指标

```bash
python experiments/wgcp_attack_eval.py \
  --input_dir data/imagenet_real \
  --glob "*.JPEG" \
  --output_dir outputs/wgcp_eval \
  --attack pgd \
  --eps 0.0313725 \
  --pgd_steps 10 \
  --pgd_alpha 0.0078431 \
  --classifier resnet50 \
  --weights_cache_dir ~/.cache/camp_torch \
  --t_star 40 \
  --self_correct_k 0 \
  --replacement_mode hard \
  --lightweight_mode \
  --save_reference_every 10 \
  --mix 0.45 \
  --clf_resize_short 256 \
  --clf_crop_size 224 \
  --min_clean_conf 0.05 \
  --max_images 100
```

建议：先用 `--max_images 6` 做连通性检查，正式对比/消融时使用 `--max_images 50~100`（推荐 100）提升统计稳定性。

## 4. 关于“分类器不能进仓库”

项目已经按这个约束处理：

1. 脚本通过 `--weights_cache_dir` 指定权重缓存目录。
2. 默认缓存目录为用户主目录下 `~/.cache/camp_torch`（不在仓库内）。
3. `.gitignore` 已加入 `models/`, `checkpoints/`, `.torch/`, `*.pt`, `*.pth`，防止大文件误提交。

## 5. 输出说明（wgcp_attack_eval）

每张图会在 `outputs/wgcp_eval/<image_name>/` 下保存：

- `00_x_clean.png`：clean 图
- `01_x_adv.png`：攻击后图
- `02_*` 到 `08_*`：WGCP 关键中间结果与最终图
- `09_compare_clean_adv_final.png`：clean/adv/final 对比
- `10_compare_adv_corrected_final.png`：adv/corrected/final 对比
- `metrics.json`：分类预测、攻击成功标记、恢复标记、图像指标

如果使用 `--lightweight_mode`，则不会生成每张图的文件夹；只会保存：

- `summary.json`
- `references/`（每 `--save_reference_every N` 张保存一张 `clean/adv/purified` 参考图）

根目录 `summary.json` 含整体统计：

- `attack_success_rate`
- `recover_rate_on_attacked`
- `clean_pred_consistency_rate`

并且会自动额外归档一份（默认在仓库外）：

- `~/.camp_runs/CAMP/wgcp_attack_eval/<timestamp>.json`
- `~/.camp_runs/CAMP/wgcp_small_scale/<timestamp>.json`

可选参数：

- `--archive_dir /abs/path/to/archive_root`
- `--archive_tag your_tag`
- `--disable_archive`（关闭额外归档）

> 注意：当前分类评估使用 clean 图预测类别作为 pseudo-label（不依赖人工标签）。用于流程验证和快速迭代非常合适；论文正式实验建议切换到有真实标签的数据集。

### 参数调优建议（针对“细节损失大/恢复率低”）

1. 当前推荐主流程：`--self_correct_k 0 --replacement_mode hard`。
2. 先调时间步：扫描 `--t_star`（例如 `20`, `30`, `40`）。
3. 若要验证 loop 是否有益，再单独设置 `--self_correct_k 1` 并扫描 `--t_bridge`（例如 `5`, `10`, `15`）。
4. 若细节仍损失大，再切 `--replacement_mode fused`，并调 `--hf_preserve/--hf_shrink`。
5. 减少占位去噪器模糊：减小 `--mix`（例如 `0.45 -> 0.3`）。
6. 评估更稳定：保持 `--clf_resize_short 256 --clf_crop_size 224`，并设置 `--min_clean_conf` 过滤低置信样本。

## 6. 当前 CTM 说明

`wgcp_small_scale.py` 与 `wgcp_attack_eval.py` 默认都使用 `GaussianCTMPredictor` 作为 CTM 占位器，目的是先验证算法机制和中间结果可解释性。

后续如果你要接入真实 CTM/LCM 权重，我可以在现有接口上直接替换 `predictor`，不改评估流程。

已提供服务器一键化配置与接入说明：

- `docs/CTM服务器一键配置与结合指南.md`
- `scripts/setup_ctm_server.sh`

并且两个实验脚本都支持：

- `--predictor_type gaussian|module`
- `--predictor_module package.module:ClassName`
- `--predictor_kwargs_json '{...}'`
- `--predictor_image_size 64`（用于 64x64 CTM 适配）
