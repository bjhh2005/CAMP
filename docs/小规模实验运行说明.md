# WGCP 小规模实验运行说明

## 1. 这份文档解决什么

用于本地快速跑通 WGCP，并给出当前推荐参数。

如果你要在 Linux 服务器接 CTM，请看：`docs/CTM服务器一键配置与结合指南.md`。
如果你要跑 A0-A5 消融，请看：`docs/消融实验方案.md`。

## 2. 环境准备（Conda）

```bash
conda env create -f environment.yml
conda activate camp
```

环境已存在时更新：

```bash
conda env update -n camp -f environment.yml --prune
```

依赖变化后回写：

```bash
# Linux
bash scripts/export_conda_env.sh camp environment.yml

# Windows PowerShell
powershell -ExecutionPolicy Bypass -File scripts/export_conda_env.ps1 -EnvName camp -OutputFile environment.yml
```

## 3. 快速运行

### 3.1 仅净化（你已有对抗图）

```bash
python experiments/wgcp_small_scale.py \
  --input_dir data/adv_samples \
  --output_dir outputs/wgcp_debug \
  --max_images 4 \
  --t_star 40 \
  --t_bridge 10 \
  --self_correct_k 1 \
  --replacement_mode hard
```

### 3.2 一体化评估（从 clean 图开始）

```bash
python experiments/wgcp_attack_eval.py \
  --input_dir data/imagenet_real \
  --glob "*.JPEG" \
  --output_dir outputs/wgcp_eval \
  --attack pgd \
  --eps 0.0313725 \
  --pgd_steps 10 \
  --pgd_alpha 0.0078431 \
  --classifier resnet50 \
  --weights_cache_dir ~/.cache/camp_torch \
  --t_star 40 \
  --self_correct_k 0 \
  --replacement_mode hard \
  --lightweight_mode \
  --save_reference_every 10 \
  --mix 0.45 \
  --clf_resize_short 256 \
  --clf_crop_size 224 \
  --min_clean_conf 0.05 \
  --max_images 100
```

建议流程：

1. 先用 `--max_images 6` 做冒烟检查。
2. 再用 `--max_images 50~100` 出稳定统计（推荐 100）。

## 4. 不同分辨率图片会不会出问题

结论：当前实现支持不同分辨率混跑，通常不会因为尺寸不一致而报错。

依据（代码行为）：

1. 每张图逐张处理（不是同 batch 拼接），因此不会出现 batch 内 shape 冲突。
2. DWT/IDWT 后会按原图尺寸回对齐：`IDWT` 重建后按原始 `H,W` 裁切，必要时补边。
3. 分类器评估前会做统一预处理（短边缩放 + 中心裁剪），所以分类阶段也不会因为原始分辨率不同而失败。

你需要注意的边界：

1. 不会报错不等于“完全公平可比”。若你要做严格横向对比，建议额外指定 `--resize H W` 统一输入尺度。
2. 如果 `--predictor_image_size` 设为 64，超高分辨率图会先缩到 64 再放回，可能导致细节损失增加；这是统计表现变化，不是流程错误。

## 5. 当前推荐参数（非消融场景）

基于现有脚本默认实践，推荐先用：

- `self_correct_k=0`
- `replacement_mode=hard`
- `ablation_ll_source=orig`
- `ablation_hard_hf_source=pred`

如果要验证 loop 价值，再单独做：

- `self_correct_k=1`
- 扫 `t_bridge in {5,10,15}`

## 6. 输出说明（wgcp_attack_eval）

默认输出 `outputs/wgcp_eval/summary.json`，包含：

- `attack_success_rate`
- `recover_rate_on_attacked`
- `clean_pred_consistency_rate`
- 每样本指标（含 pseudo-label 评估）

开启 `--lightweight_mode` 后：

1. 不再保存每张图完整中间结果目录。
2. 保存 `summary.json` + `references/`（每 N 张一张对比图）。

运行记录会额外归档到仓库外（默认）：

- `~/.camp_runs/CAMP/wgcp_attack_eval/<timestamp>.json`
- `~/.camp_runs/CAMP/wgcp_small_scale/<timestamp>.json`

可选参数：

- `--archive_dir /abs/path/to/archive_root`
- `--archive_tag your_tag`
- `--disable_archive`

## 7. 约束说明（分类器权重不进仓库）

已按约束处理：

1. 用 `--weights_cache_dir` 指定缓存目录。
2. 默认缓存在用户目录（不在仓库）。
3. `.gitignore` 已忽略 `models/`, `checkpoints/`, `.torch/`, `*.pt`, `*.pth` 等大文件。
