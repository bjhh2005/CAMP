# WGCP 小规模实验运行说明

## 1. 安装依赖

```bash
pip install -r requirements.txt
```

## 2. 准备数据

将待测试图片（当前可先把它们视作 `x_adv`）放到目录，例如：

- `data/adv_samples/0001.png`
- `data/adv_samples/0002.png`

可选：如果你有对应 clean 图，可以放在 `data/clean_samples/`，文件名与 `adv_samples` 保持一致，用于计算更可信的结构指标。

## 3. 运行（先跑阶段二，关闭自校正循环）

```bash
python experiments/wgcp_small_scale.py \
  --input_dir data/adv_samples \
  --output_dir outputs/wgcp_debug \
  --max_images 4 \
  --t_star 80 \
  --self_correct_k 0
```

## 4. 运行（开启自校正循环示例）

```bash
python experiments/wgcp_small_scale.py \
  --input_dir data/adv_samples \
  --output_dir outputs/wgcp_debug_loop \
  --max_images 4 \
  --t_star 80 \
  --self_correct_k 3 \
  --self_correct_steps 60,40,20
```

## 5. 输出说明

每张图会在 `outputs/.../<image_name>/` 下保存：

- `00_x_adv.png`：输入图
- `01_LL_orig.png`：锁定低频
- `01_HL/LH/HH_orig_vis.png`：原始高频可视化
- `02_x_t*.png`：加噪后状态
- `03_x0_hat.png`：单步预测结果
- `04_LL/HF_hat*.png`：预测图的小波分解
- `05_x_corrected.png`：低频替换后的结果
- `06_loop*.png`：自校正循环的每轮图
- `07_x_final.png`：最终输出
- `08_compare_adv_corrected_final.png`：对比拼图
- `metrics.json`：指标与参数

同时根目录会有 `summary.json` 汇总全部样本结果。

## 6. 当前实现与后续模型替换

当前 `experiments/wgcp_small_scale.py` 默认使用 `GaussianCTMPredictor` 作为 **CTM 占位器**，目的是先验证你的算法机制与中间结果保存逻辑。

如果要做论文级结果，需要替换成真实的一步一致性模型（CTM/LCM）。这一步通常会下载较大预训练权重（常见是数 GB 级），下载前建议先确定：

1. 目标数据集分辨率（例如 CIFAR-10、ImageNet 子集或自建数据）。
2. 你打算对齐的 backbone（严格 CTM 还是 LCM 近似）。
3. GPU 显存预算。

确定后我可以直接把脚本中的 predictor 接口替换成真实模型推理版，并给你对应下载命令。
